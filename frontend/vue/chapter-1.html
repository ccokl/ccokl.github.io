# 组件系统

Vue.js 的组件渲染顺序是由内而外的

# 类型

  • 页面组件（.vue)
  • 独立组件：不包含业务，独立、具体功能的基础组件
  • 业务组件

# props

  • 关于 prop 验证
  • 组件里定义的 props，都是单向数据流，只能通过父级修改，组件自己不能修改 props 的值
  • 组件可以接受任意的 attribute，这些 attribute 会被添加到这个组件的根元素上,配置 inheritAttrs: false 可以禁用

# 自定义事件

  • 自定义事件名不存在任何自动化的大小写转换
  • v-model 默认会利用名为 value 的 prop 和名为 input 的事件
  • .native 修饰,组件的根元素上直接监听一个原生事件,
  • $listeners 可以配合 v-on="$listeners" 将所有的事件监听器指向这个组件的某个特定的子元素
  • .sync对一个 prop 进行“双向绑定”

# slot

  • 承载分发内容的出口

  • 作用域：父级模板里的所有内容都是在父级作用域中编译的；子模板里的所有内容都是在子作用域中编译的。

  • 作用域插槽：让插槽内容能够访问子组件中才有的数据

# 组件的通信

  • Vuex:适用于Vue 大型项目

  • EventBus:适用于不复杂的项目

  • ref：给元素或组件注册引用信息；

  • $parent / $children：访问父 / 子实例。

  • provide / inject：主要为高阶插件/组件库提供用例,主要解决跨级组件间的通信问题(父组件向下传递数据)

  • 基于事件机制实现的 dispatch 和 broadcast 方法，用于组件件跨级通信

  • 找到任意组件实例——findComponents
