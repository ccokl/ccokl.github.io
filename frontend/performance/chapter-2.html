# 数据存储相关

# 浏览器缓存机制

缓存可以减少网络 IO 消耗，提高访问速度

获取数据优先级：

  1. Memory Cache
  2. Service Worker Cache
  3. HTTP Cache
  4. Push Cache:HTTP2 的新特性

# HTTP 缓存机制

优先级较高的是强缓存，在命中强缓存失败的情况下，走协商缓存

  • Cache-Control
  • web制作、开发人员需知的Web缓存知识

# 强缓存

  • 强缓存:若命中则直接从缓存中获取资源，不会再与服务端发生通信
  • 命中强缓存的情况下，返回的 HTTP 状态码为 200;

max-age VS s-maxage：两者同时出现时，优先考虑 s-maxage。如果 s-maxage 未过期，则向代理服务器请求其缓存内容。

public 与 private：针对资源是否能够被代理服务缓存而存在

# 协商缓存

  • 浏览器需要向服务器去询问缓存的相关信息，进而判断是重新发起请求、下载完整的响应，还是从本地获取缓存的资源

  • 命中浏览器缓存：求对应的状态码是 304；

# 本地存储

  • Local Storage、Session Storage 和 Cookie 遵循同源策略

# Cookie

  • Cookie 是紧跟域名的
  • 同一个域名下的所有请求，都会携带 Cookie
  • 过量的 Cookie 会带来巨大的性能浪费
  • 4KB
  • Cookie的设置、读取以及是否自动携带问题

# Web Storage

  • Local Storage:持久化的本地存储
  • Session Storage:临时性的本地存储
  • 5-10M
  • 位于浏览器端，不与服务端发生通信
  • 只能存储字符串

# IndexedDB

运行在浏览器上的非关系型数据库

  • 没有存储上限
  • 可以存储字符串，还可以存储二进制数据
