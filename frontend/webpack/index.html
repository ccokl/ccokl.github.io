# 模块

# CommonJS VS ES6 Module

CommonJS ES6 Module
模块依赖关系的建立发生在代码运行阶段 模块依赖关系的建立发生在代码编译阶段
值拷贝 动态映射
ES6 Module会自动采用严格模式
  • CommonJS 只有第一次会进行加载，第二次会直接从缓存取值
  • ES6 Module 可以做死代码检测和排除，但不能进行动态加载
  • ES6 Module 可以更好地支持循环依赖，只需要保证导入的值被使用时已经设置好正确的导出值

# 其他类型模块

非模块化文件：通常接口绑定在全局，正常引入即可

AMD：异步加载模块，实际应用中已经用得越来越少，大多数开发者会选择CommonJS 或 ES6 Module

UMD：通用模块标准，它的目标是使一个模块能运行在各种环境下，UMD 模块一般都最先判断AMD环境；

在Webpack中，由于它同时支持AMD及CommonJS，也许工程中的所有模块都是CommonJS，而UMD标准却发现当前有AMD环境，并使用了AMD方式导出，这会使得模块导入时出错。
