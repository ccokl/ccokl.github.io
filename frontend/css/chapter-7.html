# 7.4.2 下拉菜单淡入淡出效果

.dropdown__drawer{
  position:absolute;
  background-color:#fff;
  width:10em;
  opacity:0;
  visibility:hidden;
  transition:opacity .2s linear,visibility 0s linear .2s;
}
.is-open .dropdown__drawer {
  opacity:1;
  visibility:visible;
  transition-delay:0s;
}

由于 display 不支持过渡，所以下拉菜单淡入淡出先执行 opacity 过渡，延迟.2s在执行 visibility 过渡；visibility 可以设置元素从可见页面移除，但文档流保留，其它元素会围绕它的位置布局，下单菜单设置了绝对定位不会影响布局

# 7.5.1 变换于动画性能

思考❓

为何使用变换做动画性能会好很多？和浏览器渲染页面有关

浏览器渲染路径：布局-绘制-合成

布局：计算每个元素将在屏幕占据的空间，任何时候改变元素的宽度、位置等属性，元素位置会重新计算，浏览器会重排，重新计算其它元素布局；

绘制：填充像素：描绘文本，着色图片，边框，阴影等；在内存绘制，某些条件下元素会被单独提取一个图层交给 GPU 处理（硬件加速），图层越多需要消耗更多内存

**合成：**浏览器收集所有绘制完成的图层并把他们提取为最终显示在屏幕上的图像；合成按照特定顺序进行

当修改 opacity 和 transform 之一，浏览器会把元素提升到自己的绘制图层并使用 GPU 加速

谨慎使用 will-change 属性
