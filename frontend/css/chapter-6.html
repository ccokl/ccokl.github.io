# 6.1 模块化CSS

模块化CSS是指吧页面分割成不同的部分，这些组成部分可以在多种上下文中重复使用，并且相互之间没有依赖关系；当修改某一部分CSS时不会对其他部分产生影响。

# 6.1.3 模块样式

🌰单元素-按钮模块

.button {
  padding .5rem .8em;
  border:1px soild #265559;
  border-radius:.2em;
  font-size:1rem
}
// BEM 命名规范
.button--success{
  border-color:#cfe8c9;
  color:#fff;

.media{}
.media__image{}
.media__body {}
.media--right > .media__image {}

避免在模块选择器使用通用标签

设计模块时思考：

这个模块的职责是什么？单一职责原则

🌰模块组合--做一个下拉菜单；模块1:下拉；模块2菜单

下拉菜单

<style>
 .dropdown{
    display:inline-block;
    position: relative;
}
.dropdown__toggle{
    padding:.5em 2em .5em 1.5em;
    font-size:1rem;
    border:1px solid #ccc;
}
.dropdown__toggle::after{
    content:" ";
    position:absolute;
    right:1em;
    top:1em;
    border:.3em solid;
    border-color:#000 transparent transparent;
}
.dropdown__drawer{
    display:none;
    position:absolute;
    right:0;
    top:2.1em;
    min-width:100%;
   border-color:#eee;
  }
 .is-open .dropdown__toggle::after{
    top:.7em;
    border-color:transparent transparent #000;
  }
.is-open .dropdown__drawer{
    display:block;
}
</style>
<div class="dropdown">
  <button class="dropdown__toggle">Main Menu</button>
  <div class="dropdown__drawer">
    <ul class="menu">
      <li><a href="/">home</a></li>
    </ul>
  </div>
</div>
<script>
  (function(){
    var toggle = document.querySelector('.dropdown__toggle');
    toggle.addEventListener('click',function(event){
      event.preventDefault();
      let dropdown = event.path[1];
      dropdown.classList.toggle('is-open')
    })
  }())
</script>

1、在模块中使用定位：尽量让需要定位的元素关联到同一个模块内的其他元素

2、状态类is-；has-；开头；状态类的代码要和模块其它代码放一起，使用 JS 动态更改模块的时候要用状态类去触发改变

3、菜单模块独立于下拉模块

# 6.1.4 工具类

工具类专注于某种功能，一般只声明一次，通常放在模块样式底部

.text-center{
  text-align:center !important;
}
.float-left {
  float:left;
}
.clearfix::after,
.clearfix::before{
  content:" ";
  display:table;
}
.clearfix::after{
  clear:both;
}
.hidden{
  display:none !important;
}

工具类是最应该使用important注释的地方

# 6.3 CSS方法论

通过制定一些规则来确保代码复用和减少bug

OOCSS -- 面向对象的CSS

SMACSS -- 可扩展、模块化CSS架构

BEM -- 块、元素、修饰符

ITCSS -- 倒三角CSS

CSS in JS -- JS替代方案

BEM 介于 SMACSS 和 ITCSS 之间， SMACSS 新增布局样式部分；ITCSS 则将类别分为7个层

# 6.5 CSS优先工作流程

  1. 先看设计稿
  2. 看看模式库是否有满足
  3. 暂停页面开发，现在模式库实现新模块或者变体
  4. 回到页面开发，使用刚写的样式

优点：

  1. 为网页提供一致性友好界面
  2. 聚焦单一任务，创建更通用可复用性更好的方案
  3. 团队合理分工，对不熟悉CSS的开发者可以将工作移交经验丰富的开发者
  4. 确保文档是最新的

总结：

  • 考虑好CSS定义的API，之后不要轻易修改它
  • 使用语义版本为CSS做版本控制
  • 通常CSS 是一门“只增不减”的语言
