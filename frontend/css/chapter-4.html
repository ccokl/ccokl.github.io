# 4.1 浮动

浮动设计初衷不用于页面布局，而是浮动元素可以被拉向一侧，使文档流能够包围它（文字环绕图片），浮动元素会脱离文档流

# 4.1.1 容器折叠

浮动元素脱离文档流，其高度不会叠加到父元素上

# 4.1.2 清除浮动

  1. 将一个元素放在主容器末尾，使用 clear 属性

    .left {
  float:left;
}
.clear {
  clear:both
}
<div class="main">
	<div class="left"></div>
	<div class="left"></div>
	<div class="clear"></div>
</div>

/**末尾的 div 元素不是浮动元素，
所以主元素会包含它，通过清除浮动，主元素也包含浮动元素**/

  2. 使用伪元素::after

    /** display:table 防止伪元素的外边距折叠，（外边距无法通过单元格元素折叠）
clear 属性只针对块级元素生效 **/
.clearfix::after,
.clearfix::before {
  display:table; 
  content:" ";
}
.clearfix::after {
  clear:both; 
}

# 4.1.3 浮动陷阱

浏览器会将浮动元素尽可能地放在靠上的地方

假设每行需要放置两个浮动元素，只需要清除每行第一个元素的浮动

.media {
  float:left;
}
.media:nth-child(odd) {
  clear:left;
}

# 4.1.4 BFC

BFC 是网页的一块区域，元素基于这块区域布局，是文档流的一部分，但它将内部元素与外部的上下文隔离开来，BFC 里的元素不会跟其外部元素互相影响。

overflow:auto 创建 BFC 副作用最小

某些情况下 BFC 中内容可能会与别的 BFC 内容重叠，如内容溢出，负外边距拉升等

# 4.3.2 子元素的大小

flex: flex-grow | flex-shrink | flex-basis
flex: 1 -> flex: 1 1 0%

# 4.3.2.1 flex-basis 属性

flex-basis 定义了元素大小的基准值，初始值是auto，可以设置任意的with值，包括%，px，rem等；浏览器检查元素是否设置了with 值，有使用with值，没有就用元素自身宽度，如果 flex-basis 不是auto，忽略 with 值。

# 4.3.2.2 flex-grow 属性

基于 flex-basis 来为子元素分配剩余空间

# 4.3.2.3 flex-shrink 属性

表示子元素是否收缩防止溢出，（当初始化子元素时总体宽度会大于容器宽度）

# 4.4 Grid

CSS 网格布局可以定义自由行和列组成的二维布局，然后将元素放置到网格中

 .grid{
   display: grid;
   grid-template-columns: 1fr 1fr 1fr;
   grid-template-rows: 1fr 1fr;
   grid-gap:0.5em;
}
.grid > * {
  padding:2em;
  color:#fff;
  background-color: darkgray;
  border-radius: 0.5em;
}

# 4.5 定位

一般来说，只有在需要将元素叠放到别的元素之前时，才选用定位

position:static 默认值

position:fixed 相对于视口定位

position:absolute 相对于最近的祖先定位元素

position:relative 相对定位依赖于文档流

position:sticky（粘滞定位）的元素（沾滞定位适配所有移动设备上的浏览器，但老的桌面浏览器不支持）

# 4.5.1 层叠上下文 和 z-index

浏览器会先绘制非定位元素，然后绘制定位元素；

层叠上下文负责决定哪些元素出现在另一些元素前面

z-index 只作用于绝对定位

创建层叠上下文方式：

  • 给定位元素添加 z-index
  • 小于1的opacity属性
  • transform、filter
  • html 顶级层叠上下文

叠放顺序（从后到前）：

  • 层叠上下文的根
  • z-index为负数的定位元素（以及子元素）
  • 非定位元素
  • z-index为auto的定位元素（以及子元素）
  • z-index 为正的定位元素（以及子元素）
