# 3.1.1.1 使用构造器创建对象实例

构造器可以是普通函数，JavaScript 内置的或者宿主程序扩展的；构造器名应该声明为首字母大写的风格

在构造器函数执行的过程中传入 new 运算所产生的实例，并将该实例作为 this 对象引用传入。通过“修改或添加 this 对象引用的成员”来完成对象构造阶段的“初始化实例对象”

参数表为空等价于obj = new MyObject()

可以将构造器作为普通函数：

function foo(){
  var data = this; // 暂存this
  return {}
}
obj = new foo() // obj最终被赋值一个空对象，但并不是由 new 运算产生的实例，return 智能返回引用类型，值类型会被引擎忽略

# 3.2 JavaScript 的原型继承

"原型继承"是 JavaScript 最重要的基础模型

# 3.2.1 空（null）与空白对象（empty）

null 在 JS 中作为一个保留字存在，代表一个属性对象类型的空值，它不是空白对象；

空白对象是一个标准的，通过 Object() 构造的对象实例如：

obj = new Object()

obj = {}

# 3.2.1.1 空白对象是所有对象的基础

Object.prototype 是空白对象，构建过程可以简单的理解为“对原型的复制”

原型含义：构造器用于生成实例的模版；如果构造器有一个原型对象，则由该构造器生成的实例必然复制自该原型对象。

继承的实现：由于实例复制自 Object.prototype,所以必然有原型对象的所有属性、方法和其它性质

# 3.2.1.2 如何得倒实例对象

读写遍历

原型对象和实例对象会各自维护一张成员属性表，属性的读取规则：

  1. 保证优先读取对象的自有属性表
  2. 如果对象中没有找到读取的属性，则尝试遍历对象的整个原型链，直到原型为 null或找到属性

推论：所谓“空白对象”，是指在它的原型链上所有自由属性表都为空的对象；而所谓“原型链”就是对象所有的父类和祖先类的原型所形成的、可上溯访问的链表。

# 3.2.2 原型链的维护

原型链：对象所有的父类和祖先类的原型所形成的、可上溯访问的链表

# 3.2.2.1 外部原型链与 constructor 属性

ES6 之前的 JS 中，需要用户代码来维护一个外部原型链-构造器原型链；通过修改prototype 属性形成

function MyObject(){}
function MyObjectEx(){}
MyObjectEx.prototype = new MyObject();
/**
这会导致 MyObjectEx 拥有错误的 constructor，为了解决问题，需要手动维护原型的constructor属性
**/
MyObjectEx.prototype.constructor = MyObjectEx
/**
这样原型与父类之间的关系就被切断了，所以早期外部原型链与有效的constructor属性只能二取一
**/

# 3.2.2.2 使用内部原型链

ECMASCript 约定对象实例必须在内部持有对象的原型，并且规范了存取内部原型的方法

Object.getPrototypeOf() ：获取内部原型链

Object.setPrototypeOf() :重写内部原型

# 3.2.3 原型继承实质

对原型修改 “效果的传递”，基于两个事实：

  1. 原型：原型是一个对象
  2. 原型链：在访问属性时，如果子类对象没有，则将访问其原型的属性表

# 3.3.2.1 super 作用

填补原型继承的不足：无法调用父类方法；使用前提：明确地知道继承关系

# 3.3.2.2 super 指向什么

指向类的原型/对象的原型

# 3.3.3 类是用构造函数来实现的

传统原型继承，子类的原型总是父类的一个实例，因此子类在声明时必须先调用父类构造器；而类继承是简单的重置了原型；

在传统原型继承中 this 实例：

  • 使用当前构造器创建实例，并将该实例作为 this 引用
  • 从当前构造器开始上溯到基类

类继承中 this 实例：

  • 使用 Object() 构造器创建实例，并作为 this 引用
  • 从基类开始一直到当前构造器

类继承与传统原型继承中 this 对象的创建和使用正好相反：

  • 在类的构造方法中，不能调用 super() 之前使用 this 引用
