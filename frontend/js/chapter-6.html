# 查漏补缺

🤔普通函数和箭头函数的区别？

  1. 箭头函数本身没有自己的this，arguments，super或new.target，不能用作构造函数,箭头函数表达式对非方法函数是最合适的。
  2. 箭头函数不会创建自己的this,它只会从自己的作用域链的上一层继承this
  3. 箭头函数不绑定Arguments 对象,会引用封闭作用域内的arguments
  4. 箭头函数作为对象方法时，没有定义this绑定，会指向 window
  5. 箭头函数没有prototype属性
  6. yield 关键字通常不能在箭头函数中使用

🤔this的指向?

首先考虑清楚this出现的场景:全局作上下文，函数上下文，类上下文

  1. 在全局上下文中，不管是否严格模式，都指向全局对象
  2. 在函数上下文中，this取决于函数被调用的方式
function f1(){
  return this;
}
//在浏览器中：
f1() === window;   //在浏览器中，全局对象是window

//在Node中：
f1() === globalThis;  

function f2(){
  "use strict"; // 这里是严格模式
  return this;
}

f2() === undefined; // true

3.在类的构造函数中，this 是一个常规对象，类中所有非静态的方法都会被添加到 this 的原型中

4.派生类的构造函数没有初始的 this 绑定。在构造函数中调用 super() 会生成一个 this 绑定

5.在箭头函数中，this与封闭词法环境的this保持一致。在全局代码中，它将被设置为全局对象

6.当函数作为对象里的方法被调用时，this 被设置为调用该函数的对象

7.当一个函数用作构造函数时（使用new关键字），它的this被绑定到正在构造的新对象。

8.当函数被用作事件处理函数时，它的 this 指向触发事件的元素

9.和其他普通函数一样，类方法中的 this 值取决于它们如何被调用
