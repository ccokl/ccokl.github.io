# 2.2 声明

JavaScript 是弱类型语言，在表达式运算过程中不强制校验操作数的数据类型。JavaScript 有 6 种声明标志符方法：变量（var）、常量（const）、块作用域变量（let）、函数（function）、类（class）、模块（import）；

# 2.2.1 变量的数据类型

# 2.2.1.1 JavaScript 中的类型系统

基于 typeof 运算符，JavaScript 识别 7 种类型，如下：

值/引用类型，约定变量的使用方式，=== 运算符用来对值/引用类型的实际数据进行比较和检查；

基本类型 值/引用类型 含义 说明
undefined 值类型 未定义 未声明或声明过程中未定义
number 值类型 数值 除赋值操作外，只有数值与数值之间的运算是数值；函数/方法的返回值或者对象的属性可以是数值
string 值类型 字符串 可以直接读取指定位置的单个字符，但不能修改；字符串在赋值运算会按引用类型的方式处理
boolean 值类型 布尔值 ture/false
symbol 值类型 符号 ES6 开始支持；符号是与对象相似的值引用类型
function 引用类型 函数 JavaScript 函数存在多重含义（包括函数、方法、构造器、生成器、类以及函数对象等）
object 引用类型 对象 基于原型继承与类继承的面向对象模型

# 2.2.1.2 宿主定义的其他对象类型

在具体实现上，ECMAScript 规范接受（但不推荐） typeof 返回的 7 种类型之外的值。也就是说宿主自定的对象不能实现对象的“[[call]]"内部方法，正则表达式对象除外（兼容早期特性）

# 2.2.1.3 ECMAScript 的类型系统

ECMAScript 语言类型存在 7 种数据类型包括：Undefined、NULL、Boolean、String、Symbol、Number、Object。而 JavaScript 语言 是 ECMAScript 规范的一种实现。相当于“方言”。

ECMAScript 语言类型 VS JavaScript 语言类型

ECMAScript 语言类型 JavaScript 语言类型
语言类型首字母大写 语言类型使用字符串
Null 是一个类型，且存在唯一值 null 没有 Null 类型，null 值是对象类型的一个特殊实例
没有函数类型，函数是对象类型的一个变体 函数是第一类类型

ECMAScript 规范类型是为了实现 ECMAScript 语言类型而存在的；

ECMAScript 语言类型是为了叙述 JavaScript 语言的规范而存在的；

# 2.3.2 等值检测的相等问题

类型 运算规则
值类型与引用类型比较 将引用类型转换成与值类型相同的数据，在进行“数据等值”比较；
两个值类型比较 转换成相等数据类型的值进行“数据等值”比较；有一个是数字时将另一个转换成数字比较；有一个是布尔时它将被转换成数字，另一个也会转换成数字进行比较
连个引用类型比较 比较引用地址

JavaScript 总是尽量用数字值进行比较

# 2.3.3 等值检测的严格相等问题

类型 运算规则
值类型与引用类型比较 必然 “不严格相等”
两个值类型比较 数据类型不同，必然 “不严格相等”；否则，按值检测相等的规律比较
连个引用类型比较 比较引用地址

NAN 不等于NAN

符号可以转换为 true，但不等值于true

Symbol() === true //false

取字面量相同的引用类型，不严格相等

{} === {} //false

# 2.6 严格模式下的语法限制

"use strict"

默认处于严格模式：模块、类声明和类表达式的整个声明块、在引擎或者宿主运行参数指定

# 2.6.1 语法限制

  1. 在对象字面量声明中存在相同属性名（ES5 规范定义的，ES6 开始取消限制）
  2. 函数声明中参数表中带有相同参数名
  3. 不能声明或者重写eval 和 arguments 两个标志符
  4. 用0前缀声明八进制字面量
  5. 用 delete 删除显示声明的标志符、名称、具名函数
  6. 在代码中使用一些扩展的保留字：interface、let、private....
  7. 在代码中包括 with 语句

# 2.6.2 执行限制

  1. 向不存在的标志符赋值将导致“引用异常”
  2. 运算符处理一些不可处理的操作数时
  3. 访问 arguments.callee 或者函数的 caller 属性将导致 “类型异常”
